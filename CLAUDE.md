# Commodities Trading System

ML-based prediction system for 20 commodities with automated agents, multi-model ensembles, and portfolio optimization.

## Structure

- `coffee/`, `chocolate/`, `sugar/`, `natgas/`, `soybeans/`, `wheat/`, `copper/` — Original 7 commodities (features.py, train.py, strategy.py, fetch_*.py, data/, models/)
- `crude_oil/`, `gold/`, `silver/`, `platinum/`, `corn/`, `oats/`, `live_cattle/`, `lean_hogs/`, `cotton/`, `lumber/`, `oj/`, `heating_oil/`, `gasoline/` — 13 additional commodities
- `agents/` — Agent system with 20+ agents (see below)
- `tests/` — Test suite: `python3 -m pytest tests/ -v`
- `tools/` — Orchestration, backtesting, portfolio simulation
- `configs/` — Per-commodity training configs (generated by quality agent)
- `logs/` — Design decisions, agent observations, signals

## Key Commands

```bash
# From repo root:
python3 -m agents weekly                  # Full weekly pipeline
python3 -m agents predict                 # Generate predictions (ensemble)
python3 -m agents quality                 # Model quality diagnosis + remediation
python3 -m agents train --dry-run         # Evaluate without promoting
python3 -m agents research                # Feature stability + baselines
python3 -m agents health                  # Quick health check
python3 -m agents confidence              # Train confidence meta-models
python3 -m agents data-quality            # Validate data pipeline
python3 -m agents drift                   # Check feature drift
python3 -m agents baselines               # Compare vs simple strategies
python3 -m agents calibration             # Check prediction calibration

# Ensemble training:
python3 tools/train_ensemble.py           # Train multi-model ensembles (all)
python3 tools/train_ensemble.py natgas    # Train specific commodity

# Portfolio simulation:
python3 tools/portfolio_allocator.py      # Full portfolio backtest
python3 tools/simulate_trading.py         # Per-commodity simulation
python3 tools/volatility_strategy.py      # Volatility strategy backtest
python3 tools/accuracy_report.py          # Model accuracy dashboard

# Scaffold new commodities:
python3 tools/scaffold_commodity.py <name> # Auto-generate files

# Data fetching:
python3 tools/fetch_demand.py             # Buyer stock prices
python3 tools/fetch_macro.py              # FRED macro data (needs API key)
python3 tools/fetch_usda.py               # USDA crop data (needs API key)
python3 tools/fetch_noaa_cfs.py           # NOAA weather forecasts
```

## Model Architecture

- Multi-model ensemble: XGBoost + LightGBM + Ridge at 3 horizons (5d, 10d, 21d) = 9 models per commodity
- Per-commodity best horizon selected automatically by accuracy
- Walk-forward CV with purge gap = horizon, 252-day test folds
- Non-overlapping evaluation (honest accuracy, no autocorrelation inflation)
- Spearman rank correlation as primary metric
- Feature stability filter (drops features with non-stationary correlations)
- Regime features: HMM regime detection, GARCH volatility forecasting
- Conditional confidence meta-model: predicts when primary model is most reliable
- Model quality agent: diagnoses issues, adjusts per-commodity training configs, history-aware

## Agent Architecture

Observer agents → signal bus → model quality agent (decision-maker)
- Signal types: data_anomaly, feature_drift, calibration_off, baseline_beaten, model_degraded
- Design decisions log: logs/design_decisions.md (cross-session context)
- Claude weekly review agent sends Slack summary

## Data Sources

- Price: Yahoo Finance (yfinance)
- Positioning: CFTC Commitment of Traders
- Weather: Open-Meteo API
- Climate: NOAA ENSO indices (ONI, MEI)
